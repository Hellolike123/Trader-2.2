# Trader Pool

## Critical Rule
This skill is generated by code, not by the agent.

Run `scripts/final_pool.py <command>` and return stdout exactly. Do not handwrite, summarize, shorten, restyle, translate, or append follow-up lines after the script output.

## Natural Trigger Mapping
- `加入选股池` / `入池` / `确认入池` → `final_pool.py add --target <股票名>`
- `对比` / `比较` / `多股比较` → `final_pool.py compare --targets <股票1> <股票2> ...`
- `池内排序` / `优先级` → `final_pool.py rank`
- `作战表` / `明日计划` → `final_pool.py plan`
- `复盘选股池` → `final_pool.py review`
- `待确认池` / `pending` → `final_pool.py show-pending`

Soft phrases like `这个不错`, `可以关注`, `明天看看` must trigger `add-pending` instead of `add`.

## State
Pool data is stored outside the skill directory. Reinstalling the skill must not delete user data:
- `~/.trader/pool.json` — main pool
- `~/.trader/pending.json` — pending pool
- `~/.trader/last_plan.json` — last battle plan

## Rules
- Pool limit: 10 records. Execution pool max 3.
- Admission must be manual. `analyze` only suggests; `add` writes.
- Structure score: 缠论 45 + 威科夫 30 + 筹码 25 = 100.
- Upward momentum is a filter, not part of the total score.
- Review only checks yesterday's execution tickets from `last_plan.json`.
