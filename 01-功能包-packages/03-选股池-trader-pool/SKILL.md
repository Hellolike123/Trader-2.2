# Trader Pool

## Critical Rule
This skill is generated by code, not by the agent.

Run `scripts/final_pool.py <command>` and return stdout exactly. Do not handwrite, summarize, shorten, restyle, translate, or append follow-up lines after the script output.

This is the primary pool workflow. It does not replace the original single-stock Trader report; it consumes that style of analysis after the user confirms a stock should enter the pool.

## Natural Trigger Mapping
- `加入选股池` / `入池` / `确认入池` / `把这只加入池子` -> run `scripts/final_pool.py add --target <股票名或代码>`.
- `对比` / `比较` / `多股比较` / `帮我比一下` / `哪个更好` -> run `scripts/final_pool.py compare --targets <股票1> <股票2> ...`.
- `池内排序` / `池子优先级` / `今天优先看谁` / `候选排序` -> run `scripts/final_pool.py rank`.
- `生成明日作战表` / `生成作战表` / `明日计划` -> run `scripts/final_pool.py plan`.
- `复盘选股池` / `复盘执行票` / `复盘昨日计划` -> run `scripts/final_pool.py review`.
- `看看待确认池` / `看看推荐` / `看看 pending` -> run `scripts/final_pool.py show-pending`.
- `确认入池 <股票名>` / `把这个入池` -> run `scripts/final_pool.py confirm-to-pool --target <股票名>`.

Soft phrases such as `这个不错`、`可以关注`、`明天看看` must trigger `add-pending` instead of `add`. Tell the user to say `"看看待确认池"` to review, then `"确认入池 <股票名>"` to confirm.

## Commands
```bash
python3 scripts/final_pool.py analyze --target 南网科技
python3 scripts/final_pool.py add --target 南网科技
python3 scripts/final_pool.py add-pending --target 南网科技
python3 scripts/final_pool.py compare --targets 南网科技 中国铝业
python3 scripts/final_pool.py show
python3 scripts/final_pool.py show-pending
python3 scripts/final_pool.py confirm-to-pool --target 南网科技

```

## State
Pool state is saved outside the skill directory:

```text
~/.trader/pool.json        — 正式选股池
~/.trader/pending.json     — 待确认池（盘中先加入，盘后确认入池）
~/.trader/last_plan.json   — 上一份作战表
```

Reinstalling the skill must not delete the user pool.

## Rules
- Candidate pool limit: 10 records.
- Execution records shown in the battle plan: Top 1-2 focus, execution pool max 3.
- Admission must be manual. `analyze` only suggests; `add` writes.
- No sector data is used.
- Structure score is 100 points: 缠论 45, 威科夫 30, 筹码 25.
- Upward momentum is a filter, not part of the total score.
- Review only checks yesterday's execution tickets from `last_plan.json`.
- Use only Python standard library and the existing Trader Tencent/Sina data path.
