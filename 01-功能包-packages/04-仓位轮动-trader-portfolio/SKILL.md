# Trader Portfolio

## Critical Rule
The portfolio plan is generated by code, not by the agent.

Run the command, return the validated stdout exactly, and do not compose a shorter answer from memory. If you cannot run the script, say `Trader Portfolio skill cannot run in this environment` and include the error. Do not write a partial portfolio plan.

The current contract marker is `规则版本：trader_portfolio_v1`. If output lacks this marker, it is not current Trader Portfolio skill output.

## When to use
- User asks about `仓位`, `轮动`, `2-3只股票`, `主仓`, `副仓`, `资金怎么分`, or portfolio allocation.
- Use this for position sizing and rotation only.
- For full single-stock reports, use `trader`.
- For intraday T0 cards, use `t0-trader`.
- For candidate ranking only, use `trader-compare`.

## Workflow
1. Run `scripts/final_portfolio.py --targets <股票1> <股票2> ...`.
2. Display the script stdout verbatim as the final portfolio plan.
3. Do not add an introduction, summary, disclaimer, table, bullet list, or shortened version.
4. If one stock fails, keep the plan for the others and mark that stock as `数据失败`; if all fail, return the failure.

## Tool Requirements
- Hermes: this skill requires the `terminal` toolset. Run Hermes with terminal enabled, for example `hermes chat --toolsets skills,terminal`.
- OpenClaw: this skill requires `python3` on PATH and the ability to run shell commands.
- If terminal execution is unavailable, the skill must fail closed. Do not produce a portfolio plan from memory.

## Commands
```bash
python3 scripts/final_portfolio.py --targets 南网科技 中国铝业 三安光电
python3 scripts/final_portfolio.py --snapshot /path/to/portfolio_snapshot.json
python3 scripts/portfolio_run.py --targets 南网科技 中国铝业 三安光电 --output markdown
python3 scripts/portfolio_run.py --snapshot /path/to/portfolio_snapshot.json --output markdown
python3 scripts/portfolio_run.py --targets 688248 601600 600703 --output json
python3 scripts/validate_output.py /path/to/portfolio.md
python3 scripts/self_check.py
```

`--output json` includes validated `trader_signal_v1` objects under `signal_summaries` for downstream compare/pool/review consumers. User-facing Markdown remains the default.

## Output Contract
Default `--targets` output renders:

```text
轮动仓位 — xxx + xxx

🌍 大盘xx | 中证1000 xxx

📌 组合
    主仓  xxx  仓位 xx%
         状态：xxx
    副仓  xxx  仓位 xx%
         状态：xxx
    现金  xx%

🎯 操作
    xxx
        加仓  ...
        降仓  ...
        止损  ...

📊 仓位与信号
    xxx  每天晃 x.xx元（x%）→ 波幅偏高，首仓≤x%

🎯 关键价位
    xxx  买入 xx-xx元  防守 xx元  止损 xx元  减仓 xx元

👉
    建议...
    利弗莫尔："..."
```

No markdown tables, no 规则版本 marker, no contract version text.
Use indented alignment. ATR uses plain language (波幅偏高/偏大/正常/较低).
Do not output `ATR14=`, `极端波动`, `高波动`, `低波动`.

Snapshot mode `--snapshot` keeps existing format unchanged.
