# 关注价附近的大单确认提醒 — 设计说明

> 最后更新：2026-05-23

## 1. 目标

本功能不是做“全市场大单监控”，而是做一个面向实战决策的提醒能力：

- 当价格已经接近关注价时，如果出现符合规则的大单，及时提醒
- 盘后复盘时，回看这类大单是否真正验证了后续走势

它回答的问题是：

> 价格到位后，资金有没有开始表态？

## 2. 范围

第一版覆盖三层：

- **共享层**：统一的大单识别规则
- **`t0-trader`**：盘中关注价附近的大单确认提醒
- **`review-trader`**：盘后主力资金大单复盘

适用范围：**所有股票通用**。

## 3. 设计原则

- **保守优先**：第一版宁可少报，也不要噪音太多
- **自适应阈值**：不使用单一死阈值，按股票价格档位做基础门槛
- **关键价位绑定**：只有在接近关注价时，大单才具备提醒价值
- **共享规则**：识别逻辑只写一份，展示逻辑按 skill 分开

## 4. 核心定义

### 4.1 什么算“大单”

第一版采用“绝对金额 + 相对异常”的组合判断。

一个成交要算大单，至少满足以下条件：

1. **单笔成交额达到基础门槛**
   - 基础门槛按股价分档
   - 低价、中价、高价股票分别设置不同阈值

2. **相对近期活跃度明显放大**
   - 单笔金额高于最近一段时间的平均单笔成交额
   - 第一版不追求极端灵敏，强调“明显异常”

3. **连续同向时升级为更强信号**
   - 单笔可记为观察
   - 连续同向才进入注意或强提醒

### 4.2 什么叫“关注价附近”

关注价附近采用“双条件”判断：

1. **绝对距离**
   - 当前价与关注价的差值落在指定金额区间内

2. **相对距离**
   - 当前价与关注价的偏离比例落在指定百分比阈值内

两者结合，避免低价票和高价票被同一把尺子误判。

## 5. 提醒等级

第一版采用三档，并默认偏保守：

### 5.1 观察

- 已接近关注价
- 只出现单笔异常
- 没有连续性，或方向尚不明确

### 5.2 注意

- 已接近关注价
- 连续出现同向大单
- 或者大单方向开始与价格方向一致

这是第一版最核心的提醒档位。

### 5.3 强提醒

- 连续性明显
- 强度超过阈值很多
- 与价格方向高度一致

用于真正明显的资金行为，不作为常规提示。

## 6. `t0-trader` 设计

`t0-trader` 负责盘中实时提醒，语义是：

> 价格先到关注价附近，再看有没有大单确认。

### 输出目标

嵌入现有 T0 分析结果中，给出简洁提醒，不单独做重型看板。

### 建议输出内容

- 当前是否接近关注价
- 是否出现大单
- 大单方向是承接还是砸盘
- 当前提醒等级：观察 / 注意 / 强提醒
- 建议动作：继续观察 / 提高警惕 / 重点盯盘

## 7. `review-trader` 设计

`review-trader` 负责盘后复盘，语义是：

> 今天或某段时间，主力资金到底有没有介入，怎么介入，最终有没有被走势验证。

### 输出目标

总结资金行为与价格响应，而不是做盘中打扰。

### 建议输出内容

- 今日大单分布摘要
- 大单主要发生的时间段
- 资金方向：承接 / 扫货 / 砸盘 / 派发
- 大单出现后价格的响应
- 复盘结论：有效 / 无效 / 背离

## 8. 共享模块职责

共享层只负责规则，不负责展示。

应包含：

- 大单定义
- 自适应阈值计算
- 连续同向识别
- 大单强度统计
- 标准化结果输出

`t0-trader` 和 `review-trader` 只消费这些结果并做各自的解释。

## 9. 第一版落地顺序

推荐实现顺序：

1. **共享大单识别规则**
2. **接入 `t0-trader` 的关注价附近提醒**
3. **接入 `review-trader` 的盘后复盘总结**
4. **后续再加成交活跃度修正与更细分的资金画像**

## 10. 成功标准

第一版成功的标志不是“提示越多越好”，而是：

- 在价格接近关注价时，能及时发现真正值得注意的大单
- 盘中提醒不过度打扰
- 盘后复盘能解释资金行为与走势之间的关系
- 同一套识别规则能被 `t0-trader` 和 `review-trader` 复用

## 11. 备注

- 第一版不做全市场大单雷达
- 第一版不追求极端灵敏
- 第一版优先保证可解释性和实战可用性
